# 介绍
流程引擎是专门用来处理 WMS 大型协作流程的框架特性。

流程引擎擅长解决：
- 设备、接口、人员操作的流程很复杂；
- 某些流程（比如审批流）经常调整；

工作流不适用于:
- 如果工作节点之间需要频繁转换换数据，不适用；
- 不擅长细节性的流程描述，他带来的性能损耗甚至会得不偿失；
- 不擅长处理性能要求极高的场景；

流程引擎是借助数据库完成的，使用工作流时，每个流程步骤可能都会存储进数据库，因此性能方面会有一定损耗（经测试 每秒事务总吞吐量<=3000TPS）；
